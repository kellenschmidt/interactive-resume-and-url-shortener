Summary

Wikibooks Search Engine is an inverted file index search engine for the Wikibooks database with TF/IDF relevancy ranking and boolean query processing written in C++.

What I did

Worked with 2 other students to build the application:

  • Designed an index structure to quickly and concisely process 1GB of data and store the necessary information
  • Created and optimized custom AVL tree and hash table data structures
  • Wrote a query processor to search the more 200,000 XML entries in the index

Features

  • Can parse into one of 2 different underlying data structures: AVL tree and hash table
  • Parses entire corpus into hash table in under 30 seconds or into AVL tree in under 7 minutes
  • Graphical User Interface to add and remove documents, query the index, and view search results
  • Can generate and read from a local persistent index
  • Capable of processing boolean statements in the search query
  • Ranks search results according to TF/IDF relevacny
